# Security Policy

## Supported Versions

Only the latest released version of this project is supported with security
updates.

Older versions may receive fixes at the discretion of the maintainers but
should not be relied upon for security updates.

---

## Reporting a Vulnerability

Please do **not** report security vulnerabilities through public GitHub issues.

To report a vulnerability, open a **private GitHub security advisory** for this
repository.

When reporting, please include:
- A clear description of the vulnerability
- Steps to reproduce, if applicable
- Potential impact or severity
- Any relevant logs or proof-of-concept details

You can expect an initial response within a reasonable timeframe.
If the vulnerability is confirmed, a fix or mitigation will be coordinated and
disclosed responsibly.

---

## Disclosure

Security issues are handled in accordance with responsible disclosure
practices. Public disclosure will occur only after a fix or mitigation is
available, or when deemed appropriate.
